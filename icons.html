§ 12 · icon system

28 icons.
One grammar.

Every icon in the ZopDev system is drawn on a 24×24 grid with a 1.75px stroke, square caps, and miter joins. No rounded corners. No duotone. Monochrome via currentColor so they inherit from the surrounding text. Click any icon to copy its <use> reference.

§ 01 · arrows

Arrows + chevrons.

Four directions, two weights. Use chevrons for affordance (accordion, nav trail), arrows for action (CTA, go-to).

i-arrow-rightcopied
i-arrow-up-rightcopied
i-chevron-rightcopied
i-chevron-downcopied

§ 02 · nav & ui

Nav & interface.

Search is a rectangle + handle, sharp corners. External-link uses a solid page with a stroke arrow breaking out, communicates "leaves this domain".

i-externalcopied

§ 03 · state

Status icons.

The ok / warn / err triad uses its CSS tint colors. i-dot-live is a solid square block, pair it with the live-pulse animation from tokens.css.

§ 04 · clouds

Cloud providers.

Geometric abstractions, not vendor trademarks. For partner-tier logos (actual AWS, Azure, GCP marks), use official vendor assets per their brand guidelines. These are internal symbols for table rows and filter chips.

§ 08 · zop

The mark as icon.

i-zop-mark is the only colored icon, three tiles forming the signature L-shape. Use exactly where the full logo is too much but branding is needed (favicon fallback, compact rows, inline references in copy).

i-zop-markcopied

§ 09 · sizes

Four sizes. Nothing in between.

Pick a size from the ladder, don't invent 19px or 22px.

14px · inline
18px · table row
22px · ui default
32px · card header
48px · hero

§ 10 · coloring

Tint via currentColor.

All icons use stroke:currentColor or fill:currentColor. Change their color by changing the surrounding text color.

§ 11 · rules

Do · Don't.

DO
  • Draw on the 24×24 grid · stay within it
  • 1.75px stroke · square caps · miter joins
  • Tint with CSS currentColor; inherit from text
  • Stick to the 4-size ladder: 14 · 18 · 22 · 32
  • Use status tints (ok/warn/err) for state icons
  • Use fill:currentColor for solid-block glyphs (filled class)
DON'T
  • No rounded corners · no stroke-linecap:round
  • No duotone · no gradients · no shadows
  • No invented sizes (19px / 26px / 40px)
  • No hardcoded fills, always currentColor
  • No custom icons outside this sprite without a PR
  • No redrawing the Zop mark at icon size, use i-zop-mark

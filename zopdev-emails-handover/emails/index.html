Engineering notes for the dev

  • Self-contained. Logo is base64-embedded (~29 KB per file). No asset hosting required. Swap to a CDN URL only if Outlook desktop is critical — some versions strip data URIs.
  • Table-based layout. Outlook 2007–2019 use Word’s HTML engine; flexbox/grid don’t render. Classic <table> nesting is the safe path.
  • MSO conditional VML buttons. Each CTA has a <v:roundrect> fallback wrapped in <!--[if mso]> for Outlook desktop button fidelity. The non-MSO branch is the modern HTML/CSS button for everywhere else.
  • Dark mode. Applied via @media (prefers-color-scheme: dark) for Apple Mail / iOS Mail / Outlook.com modern, plus [data-ogsc] attribute fallback for legacy Outlook.com. Gmail web is partially supported.
  • Plain-text companion. Every file has a plain-text version in an HTML comment at the bottom. Most ESPs auto-generate this, but audit it — clean plain-text helps deliverability and is what shows in Apple Watch / smartwatch previews.
  • Merge fields use Mustache-style {{variable}}. Swap to your service’s syntax: SendGrid {{variable}}, Postmark {{variable}}, Resend {{variable}}, Mailgun %recipient.variable%, etc.
  • For the showcase / visual review: the carousel page lives at ../email-showcase.html. The reusable skeleton without copy is at ../email-template.html.

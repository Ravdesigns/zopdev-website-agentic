§ 18 · motion system

Motion is the product, working.

Not decoration. Not polish. Every animation in the ZopDev system exists to signal that something is happening, telemetry you can watch. The page isn't static; the product isn't static. That's the whole philosophy, in one sentence.

§ 01 · philosophy

Three principles.

01 · functional

Every motion signals.

If an animation doesn't carry meaning, tell the user something happened, is happening, or is reversible, it doesn't ship. No entrance flourishes. No decorative transitions.

02 · hard offset

Sharp over soft.

Motion mirrors the visual language: hard offset shadows, not soft blur. Card lift = 4px up + 8px offset block, not a 20px gaussian shadow. Sharp corners all the way.

03 · respectful

Reduced-motion always wins.

One global rule collapses every transition to 1ms when the user’s OS says so. The loader skips entirely. Toast still cycles but without the slide. Never ask permission twice.

§ 02 · durations

Four. No more.

Four durations cover every animation the system does. If you need a 350ms or a 600ms, the design is wrong, pick one from the ladder.

tokenmswhere it’s used
--dur-fast160msHover state changes, arrow slide, color-only transitions. Feels instant but perceptible.
--dur-med250msCard lift, border brighten, shadow offset, CTA state. The signature duration.
--dur-slow400msThree-numbers count-up, tile color-fill, hero content entrance. Slow enough to feel deliberate.
--dur-xslow520msSection reveal on scroll. One use-case only. Any slower feels broken; any faster misses the reveal.

§ 03 · eases

Four cubic-beziers.

Four curves. Each has a specific job. If you're reaching for ease-in or linear, stop, none of these are in the system.

--ease-out cubic-bezier(.22, 1, .36, 1) default for card lift, hover, reveal
--ease-in-out cubic-bezier(.65, 0, .35, 1) subtle loops: mark pulse, favicon breathe
--ease-spring cubic-bezier(.175, .885, .32, 1.275) overshoot: loader piece drop, bouncy entrances
--ease-snap cubic-bezier(.19, 1, .22, 1) close-in accelerations, modal open

The card-lift.

One hover behavior defines the whole system. Every card, tile, surface, button, row uses it. Hover any card below to feel the move.

§ 07 · loader

The Tetris brand intro.

First-visit-per-session page-load: a 2×2 tile piece drops from above, rotates 90°, lands with a spring bounce, tiles color-fill in sequence (blue → orange → green), the wordmark reveals letter-by-letter from the left, and the whole overlay slides up. Full sequence: ~2.9 seconds. Gated by sessionStorage so it only runs once per session, and by prefers-reduced-motion so it auto-skips.

§ 08 · accessibility

One rule, everywhere.

Tokens.css declares a global reduced-motion rule that collapses every transition and animation in the system. You don't need to remember it per-component, it's already there.

⚠ required

Any new animation must respect this rule.

If you add a new keyframe or transition in a page-specific <style> block, the global rule in tokens.css will already collapse it. But if you're animating via JavaScript (e.g., requestAnimationFrame), you must check matchMedia('(prefers-reduced-motion: reduce)').matches yourself and no-op.

§ 09 · rules

Do · Don’t.

DO
  • Use only the four durations and four eases defined in tokens.css
  • Card-lift signature (translateY(-4px) + 0 8px 0 -4px pillar) for any hover
  • Gate every animation with prefers-reduced-motion
  • Use ease-spring only for overshoot moments (loader, first-entry)
  • Stagger sequential entries by 70-240ms depending on count
  • Make the toast content rotate, never a static loop
DON'T
  • No spinning loaders · no infinite wheels · no CSS spinners
  • No custom ease curves · no linear · no ease-in
  • No drop-shadow with blur radius on hover
  • No opacity-fade-only reveals (always pair with transform)
  • No animations > 600ms except section reveal (520ms)
  • No hover-triggered animations on touch devices (@media (hover:none) disables)
